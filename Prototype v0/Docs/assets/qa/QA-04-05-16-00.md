## 1. Cenário de Teste
- **Perfil Selecionado:** Estudante (Graduação)
- **Dados:** Nome: "Student QA Test", Email: "student_qa@ariano.test"
- **Ambiente:** `https://projetoariano.vercel.app/cadastro`

## 2. Erros Identificados (Bugs)

### 🔴 Bug 1: Corrida de Dados (Race Condition) no Cadastro
- **Descrição:** A animação de IA iniciava antes mesmo do backend retornar o ID do novo usuário. 
- **Consequência:** O componente tentava enviar `entity_uid: null` para o backend, resultando em erro **422 Unprocessable Entity**.
- **Impacto:** A animação travava logo no início com a mensagem "Erro de Processamento".

### 🔴 Bug 2: Timeout de Registro no Vercel (504)
- **Descrição:** O processo síncrono de registro + extração inicial de PDF estava excedendo os 60s em alguns casos.
- **Consequência:** O usuário não era criado a tempo, gerando erros de autenticação subsequentes.

### 🔴 Bug 3: TypeError no Frontend
- **Descrição:** O frontend tentava ler a propriedade `context` de uma resposta de erro, causando crash silencioso no console.

## 3. Correções Aplicadas (Hotfix e12983d)

1.  **Sincronização Atômica**: O componente `CognitionExperience` agora possui uma trava: ele **aguarda obrigatoriamente** que o `userId` seja diferente de nulo antes de disparar a primeira chamada de IA.
2.  **Otimização de Timeout**: Aumentado o `maxDuration` do Vercel para 60s (limite máximo Hobby/Pro) para suportar a latência das LLMs da Nvidia.
3.  **Tratamento de Erros Robusto**: Adicionadas verificações `if (res.status === 'error') throw new Error(...)` em todas as etapas da pipeline.

## 4. Próximos Passos (Recomendação)
- **Configuração de Neo4j Real**: O sistema ainda reporta "Modo Memória" em produção se as variáveis de ambiente não estiverem no painel do Vercel. Isso causará perda de dados entre requisições.
- **Validação com Usuário**: Solicitar que o usuário realize um novo teste agora que a trava de `userId` está ativa.

---
**Resultado Esperado do Fix:** A animação deve agora aguardar o sucesso do registro (spinner de "Finalizar") e só então iniciar a jornada visual com o UID correto.
